# 강의 자료 변경 체크리스트

## 작업 전

- [ ] 목적과 변경 범위를 한 문장으로 정리했다.
- [ ] 수정할 원본 파일을 확인했다.
- [ ] PC·모바일·공개 링크 중 갱신 대상을 정했다.
- [ ] 변경하지 않을 항목을 확인했다.
- [ ] 완료 조건을 확인했다.

## 콘텐츠

- [ ] 한 슬라이드에 핵심 주장이 하나다.
- [ ] 요청하지 않은 통계나 기능을 추가하지 않았다.
- [ ] 문장이 자연스럽고 발표자가 읽기 쉽다.
- [ ] 실제 업무 예시가 포함됐다.

## PC

- [ ] 16:9 화면에서 제목과 본문이 잘리지 않는다.
- [ ] 새 슬라이드의 순서가 자연스럽다.
- [ ] 워터마크가 내용을 과도하게 가리지 않는다.

## 모바일

- [ ] 9:16 화면에서 확대 없이 읽힌다.
- [ ] 본문이 너무 작지 않다.
- [ ] 문장이나 카드 중간에서 페이지가 잘리지 않는다.
- [ ] 워터마크가 하단 전용 여백에 있다.

## 산출물

- [ ] `slides.html`이 최신이다.
- [ ] PC PDF가 최신이다.
- [ ] 모바일 PDF가 최신이다.
- [ ] `downloads/index.html`의 페이지 수와 설명이 맞다.
- [ ] 전체 문구 수정용 파일이 최신이다.

## Git·배포

- [ ] `git status -sb`를 확인했다.
- [ ] 관련 파일만 스테이징했다.
- [ ] `tmp/`, QA 이미지, 임시 스크립트가 커밋에 포함되지 않았다.
- [ ] `git diff --cached --stat`를 확인했다.
- [ ] 공개 URL이 HTTP 200으로 열린다.
- [ ] 캐시를 피한 주소에서 새 버전을 확인했다.
