🔬 심화 — LLM 내부를 한 단계씩 (전문 과정 시각화)

위 5단계를 더 깊이 열어보면 이렇게 동작합니다. 조금 전문적이지만 그림으로 보면 쉬워요.

STEP 1 · TOKENIZATION

① 토큰화 — 문장을 "조각(토큰)"으로 + 번호 매기기

AI는 글자를 그대로 못 읽습니다. 자주 쓰는 덩어리(서브워드)로 쪼개고, 각 조각에 사전 번호(ID)를 붙입니다.

인공#8123 지능#4502 은#11 똑똑#9981 하다#205

"인공지능은 똑똑하다" → 5개 토큰 → 숫자 ID 배열 [8123, 4502, 11, 9981, 205]

STEP 2 · EMBEDDING

② 임베딩 — 각 토큰을 "의미 벡터(숫자 목록)"로

번호만으로는 의미가 없죠. 각 토큰을 수백 개 숫자(벡터)로 바꿉니다. 이 숫자들이 "의미 좌표"예요. 비슷한 뜻은 가까운 좌표에 모입니다.

지능 #4502▶
…실제론 768~수천 개 숫자

의미 공간에서는 비슷한 단어끼리 모입니다:

👑 왕 👑 여왕 🍎 사과 🍌 바나나 🐱 고양이 🐶 강아지

'왕·여왕'은 가깝고 '사과·바나나'는 따로 모입니다. 그래서 AI가 "비슷한 의미"를 이해합니다. (왕−남자+여자 ≈ 여왕 같은 계산도 가능)

STEP 3 · POSITIONAL ENCODING

③ 위치 정보 더하기 — 순서를 기억

"개가 사람을 물다"와 "사람이 개를 물다"는 단어가 같아도 뜻이 정반대죠. 그래서 각 토큰에 "몇 번째인지" 위치 정보를 더합니다.

개가pos 1→ 사람을pos 2→ 물다pos 3

위치가 바뀌면 의미가 바뀜 → 순서 정보는 필수입니다.

STEP 4 · SELF-ATTENTION (핵심)

④ 어텐션 — 각 단어가 "어디에 주목할지" 계산

트랜스포머의 핵심. 각 단어가 문장 속 다른 단어 중 무엇이 중요한지 가중치를 매깁니다. 아래는 '주목도' 표 — 밝을수록 더 주목합니다.

나는은행돈을찾았다 나는 은행 돈을 찾았다

'찾았다'는 '돈을·은행'을 강하게 봅니다 → 문맥(돈을 인출)을 이해. 이렇게 단어 사이 관계를 파악합니다.

STEP 5 · TRANSFORMER LAYERS

⑤ 층층이 반복 — 의미를 점점 정교하게

②~④ 과정을 수십~수백 층 쌓아 반복합니다. 아래에서 위로 갈수록 더 추상적이고 정교한 의미를 잡아냅니다.

신호가 아래(입력)에서 위(출력)로 흐르며 층마다 의미가 다듬어집니다.

STEP 6 · OUTPUT (SOFTMAX)

⑥ 다음 토큰 확률 만들기 → 하나 고르기

마지막 층의 결과를 전체 사전 단어에 대한 "확률"로 바꿉니다(softmax). 그중 하나를 골라 출력하고, 다시 ①로 돌아가 반복(자기회귀).

똑똑하다64%
유용하다21%
빠르다11%

'temperature(온도)'를 높이면 과감하게, 낮추면 안정적으로 단어를 고릅니다.

API = 프로그램끼리 주문을 주고받는 "약속된 창구"

손님이 직접 주방에 들어가지 않고 웨이터를 통해 주문하고 음식을 받죠. API가 바로 그 웨이터입니다. 요청·응답이 계속 오갑니다.

손님은 주방 사정을 몰라도 됨 → 정해진 방식으로 요청하면 결과가 옴.

그래서 자동화 툴이 만들어집니다

1️⃣ 내 툴이 엑셀에서 데이터를 읽음
2️⃣ API로 AI에 요청 "이 메모 요약해줘"
3️⃣ AI가 답을 돌려줌
4️⃣ 그 답을 다시 엑셀 셀에 자동 입력
5️⃣ 1,000행 반복 → 사람 없이 자동

버튼 한 번으로 "읽기 → AI 처리 → 저장"이 도는 게 자동화 툴의 정체.
