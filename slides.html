01LLM 내부 ①

토큰화 → 임베딩 → 의미 공간

문장을 조각(토큰)으로 나누고, 각 조각을 의미 좌표(벡터)로 — 비슷한 뜻은 가까이 모입니다.

① 토큰화 — 조각 + 번호

인공#8123 지능#4502 은#11 똑똑#9981 하다#205

"인공지능은 똑똑하다" → ID 배열 [8123, 4502, 11, 9981, 205]

② 임베딩 — 의미 벡터

지능 #4502 ▶

…실제론 768~수천 개 숫자. 이 숫자들이 곧 "의미 좌표"

👑 왕 👑 여왕 🍎 사과 🍌 바나나 🐱 고양이 🐶 강아지
'왕·여왕'은 가깝고 '사과·바나나'는 따로 모임 → AI가 "비슷한 의미"를 이해. (왕−남자+여자 ≈ 여왕 같은 계산도 가능)
01LLM 내부 ②

위치 → 어텐션 → 층 → 출력

④ 어텐션 — 어디에 주목할지 (핵심)

밝을수록 더 주목 — "찾았다" → "돈을·은행"을 봄

나는은행돈을찾았다 나는 은행 돈을 찾았다

⑤ 트랜스포머 층

아래→위로 의미가 점점 정교해짐

⑥ 출력 (Softmax)

다음 단어 확률 → 하나 선택

똑똑하다64%
유용하다21%
빠르다11%

③ 위치 인코딩 — 순서를 기억

"개가 pos1 사람을 pos2 물다 pos3" ≠ "사람이 개를 물다" — 위치가 바뀌면 의미가 정반대. 그래서 각 토큰에 순서 정보를 더합니다.

④ 어텐션이 트랜스포머의 핵심 — 각 단어가 문장 안 다른 단어에 얼마나 "주목"할지 계산해 문맥을 이해.
01LLM 학습

같은 질문, 학습 단계별로 답이 변한다

입력: "오늘 기분이 어때?" — 단계마다 답이 어떻게 발전하는지

① 사전학습 직후📚 인터넷 글로 다음 단어 채우기만

🤖 "오늘 기분이 어때? 그래, 가끔 사람들은 자기 기분에 대해 묻곤 하지. 옛날 그리스 철학자들은…"

⚠️ 질문에 답한다는 개념 X — 그저 글 이어 쓰기 (자동 작문기)

② 지도 미세조정(SFT) 후🧑‍🏫 좋은 질문·답변 예시 모방

🤖 "안녕하세요. 저는 AI 언어 모델이므로 감정을 가지고 있지 않습니다. 도움이 필요하시면 알려주세요."

✓ 답변 형식은 익혔지만 딱딱·사무적

③ 인간 피드백(RLHF) 후👍 사람이 좋다고 평가한 답으로 강화

🤖 "고마워요, 그렇게 물어봐 주셔서 😊 저는 감정은 없지만 오늘도 도움이 될 준비는 됐어요! 본인 기분은 어떠세요?"

⭐ 자연스럽고 따뜻 + 되묻기 (우리가 쓰는 AI는 ③번까지 완료)

06API 데이터 흐름

값이 어떻게 전송·저장·수정·답변되는가

연락처 등록 예 — JSON으로 값이 그대로 오감

🙋 📤 POST /api/contacts
{ "name": "김철수", "phone": "010-1234-5678" }
💾 DB → INSERT
contacts 테이블 → 새 행 id = 1042 name = "김철수" phone = "010-1234-5678"
🙋 ✏️ PUT /api/contacts/1042
{ "phone": "010-9999-1111" }
🤖 📥 GET → 200 OK
{ "id": 1042, "name": "김철수", "phone": "010-9999-1111", "updated": "2026-06-03 14:25" }
POST(생성)→INSERT→PUT(수정)→UPDATE→GET(조회). 값이 JSON으로 오가니 어떤 화면·앱이든 같은 데이터 사용 — 자동화의 토대.
